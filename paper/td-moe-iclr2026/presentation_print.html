前言（上）：大模型的尺度與 MoE 的動機

近年語言模型參數規模與上下文長度急遽上升——dense Transformer 幾乎每顆權重、每層都參與每次前向；訓練與推理的算力與記憶體需求跟著線性甚至超線性惡化，難以在單卡／邊緣部署。

為什麼要 MoE？

  • Scaling law：更大模型通常帶來更好能力，但 dense 路線讓每個 token都負擔接近全模型的成本。
  • MoE 的核心交換：用更多專家參數換取總容量，再以稀疏門控讓每次只啟用 top-K，把「單步算力」壓在可接受範圍。
  • 直覺比喻：像圖書館從「每位讀者翻遍整座館」改成「館員只開幾個相關書櫃」——讀到的知識總量可增，但當下搬動的書變少。
銜接下一頁：MoE 緩解了「算力與參數規模」矛盾，卻讓所有專家權重仍須常駐記憶體——下一頁說明這個部署瓶頸。

核心矛盾

MoE 的效率承諾：每個 token 只路由到 top-K 個專家（如 Mixtral 啟用 2/8），計算量僅為 dense 模型的一小部分。

但記憶體代價：即使每次只啟用 2 個專家，全部 8 個專家的權重都必須常駐 GPU 記憶體——因為路由是動態的，無法預知下一個 token 會用哪個專家。

問題有多嚴重？

  • 專家權重佔模型總參數 >80%
  • NVIDIA A100 80GB 僅夠裝一個 Mixtral（FP16）
  • 消費級 GPU（RTX 4090 24GB）完全無法載入
  • 多 GPU 並行則拉高硬體成本與通訊開銷
結論：MoE 的記憶體瓶頸來自「全專家常駐」，壓縮專家權重是部署的必經之路。

既有方法的侷限：見樹不見林

傳統 SVD：逐專家壓縮

  • MoE-SVD：對每個 \(W^{(i)}\) 獨立做 SVD → \(U_i\Sigma_i V_i^\top\)，截斷前 \(r\) 個奇異值
  • 每棵「樹」壓縮得很精細，卻看不到整片森林

錯失的結構性冗餘

同一層中 64 個專家（如 Qwen2）被餵入相似的 token 分佈，權重矩陣之間存在大量共享子空間——逐專家 SVD 把它們當作毫無關聯的獨立個體。

高壓縮比的崩潰數據：
Mixtral 60% 壓縮 → MoE-SVD Avg 從 0.63 掉到 0.37（−41%）
Phi-3.5 60% → SVD-LLM PPL 爆衝到 7168（原始 3.5）

問題定義：校正集上的行為保留

一層 \(K\) 個專家 \(W^{(i)}\!\in\!\mathbb{R}^{d_{\text{out}}\times d_{\text{in}}}\)。壓縮目標——在校正分佈上最小化輸出誤差：

\[\min_{\{\hat W^{(i)}\}}\;\mathbb{E}_{x\sim\mathcal{D}_{\text{calib}}}\!\left[\sum_{i=1}^{K}\big\|W^{(i)}x-\hat W^{(i)}x\big\|_2^2\right]\tag{1}\]
破解校正目標（Equation 1）：這其實就是 MSE。
標準 MSE
\[\mathrm{MSE}=\mathbb{E}\big[\|y-\hat y\|_2^2\big]\]
本論文 Equation (1)
\[\mathbb{E}_{x\sim\mathcal D_{\text{calib}}}\!\left[\sum_{i=1}^K\|W^{(i)}x-\hat W^{(i)}x\|_2^2\right]\]

對照關係：\(y\!\leftrightarrow\!W^{(i)}x\)、\(\hat y\!\leftrightarrow\!\hat W^{(i)}x\)；只是在 MoE 場景下再加上「跨 \(K\) 個專家求和」。

對同一筆輸入 \(x\)，比較原始專家輸出 \(W^{(i)}x\) 與壓縮後輸出 \(\hat{W}^{(i)}x\)，兩者做 \(L_2\) 距離平方，對所有專家加總後，再對校正資料分佈取期望。

也就是把「原模型行為」當教師訊號：誤差越小越好， 代表壓縮模型在校正集上的行為越接近原模型。

  • 不依賴 label；純 post-training，校正集僅 256 筆
  • 壓縮後無需微調，直接部署

核心原理：從 SVD 到 Tucker 分解

回顧：矩陣 SVD（二階）

1 給定矩陣 \(W\!\in\!\mathbb{R}^{m\times n}\)，SVD 分解為：

\[W = U\Sigma V^\top \approx U_r\Sigma_r V_r^\top\]

保留前 \(r\) 個奇異值 → 參數從 \(mn\) 降為 \(r(m+n)\)。

推廣：三階 Tucker 分解

2 給定張量 \(T\!\in\!\mathbb{R}^{d_1\times d_2\times d_3}\)，Tucker 分解為：

\[T \approx G \times_1 U^{(1)} \times_2 U^{(2)} \times_3 U^{(3)}\]

\(G\!\in\!\mathbb{R}^{r_1\times r_2\times r_3}\) 為核張量，\(U^{(n)}\!\in\!\mathbb{R}^{d_n\times r_n}\) 為因子矩陣。

3 Mode-n 乘積定義（元素層面）：

\[(X\times_n U)_{i_1\cdots j\cdots i_N}=\sum_k X_{i_1\cdots k\cdots i_N}\,U_{kj}\]

沿第 \(n\) 維做矩陣乘法，其餘維度不動。

元素層面重建公式

\[T_{i_1 i_2 i_3}\approx\sum_{j_1,j_2,j_3}G_{j_1 j_2 j_3}\,U^{(1)}_{i_1 j_1}U^{(2)}_{i_2 j_2}U^{(3)}_{i_3 j_3}\]

為何比 SVD 更好？（直覺）

  • SVD 只壓二維：對每個 \(W^{(i)}\) 獨立壓（input × output），不碰專家維度
  • Tucker 壓三維：\(U_1\) 從 \(K\) 個專家中提取 \(r_1\) 個「元專家」基底，\(U_2,U_3\) 壓輸出/輸入子空間
  • 交互項：核張量 \(G\) 捕捉三個維度之間的交互結構——這是逐專家 SVD 根本無法觸及的

SVD 是 Tucker 的特例

當 \(d_1\!=\!1\)（只有一個專家）時，Tucker 退化為二階 SVD。也就是說 SVD 是 Tucker 的子集——Tucker 嚴格更強。

為何 Tucker 能壓更多？理論參數計算

以 Mixtral-8×7B（K=8, dout=14336, din=4096）一層 FFN 為例，對比三種壓縮策略的參數量差距。

① 原始權重 \[P_{\text{orig}} = K \cdot d_{\text{out}} \cdot d_{\text{in}} = 8\!\times\!14336\!\times\!4096 \approx \mathbf{470\text{ M}}\]
② 逐專家 SVD（各自獨立） \[P_{\text{SVD}} = K\cdot r\cdot(d_{\text{out}}+d_{\text{in}}) = 8\cdot r\cdot 18432\]

目標 50% 壓縮 → r ≈ 1596；仍有 8 份完全獨立的 \(U_i, V_i\)，跨專家冗餘全未利用。

③ Tucker 聯合分解 \[P_{\text{Tucker}} = \underbrace{r_1 r_2 r_3}_{\text{核張量}} + \underbrace{K r_1}_{\approx0} + \underbrace{d_{\text{out}} r_2}_{\text{共享}} + \underbrace{d_{\text{in}} r_3}_{\text{共享}}\]

取 r₁=6, r₂=2048, r₃=3072：

\[= 37.7\text{M} + 48 + 29.4\text{M} + 12.6\text{M} \approx \mathbf{79.7\text{ M}}\]

壓縮率 83%！且 Kr₁ = 8×6 = 48 ≈ 0，幾乎免費的「跨專家壓縮」。

直覺：SVD 需要 8 組各自完整的基底；Tucker 只需 r₁=6 個元專家基底，輸入/輸出子空間全專家共享。

TD-MoE 完整管線（Algorithm 1）+ 視覺化流程

核心策略一句話：在壓縮（分解）模型權重之前，先對資料進行「去相關」與「尺度平衡」的預處理，確保壓縮演算法不會被誤導。壓縮完畢後再將預處理的逆轉換吸收進因子矩陣，推論時完全無額外開銷。

為什麼需要「白化」？—— 病態特徵空間的陷阱

問題：直接分解原始權重效果不好

模型運作時，輸入特徵（activations）之間往往具有高度的相關性。這使得特徵空間處於「病態（ill-conditioned）」狀態：

  • 共變異矩陣的特徵值跨越近 4 個數量級（如 Layer 9：9 → 54,000）
  • 某些方向被極度放大，某些被壓扁 → 分解出的奇異值無法真實反映該特徵對模型輸出的實際貢獻
  • 直接依據這些數值截斷 → 可能丟棄實際重要的資訊

多線性白化：四步驟詳解

將傳統 2D 白化擴展到 3D 張量環境（因此稱為「多線性」），利用真實資料統計量引導分解。

1 收集真實數據統計量

使用校準數據（256 筆 WikiText-2）讓模型跑一次 forward + backward：

  • 收集輸入激勵值 \(X \in \mathbb{R}^{N \times d_{\text{in}}}\)
  • 收集輸出梯度 \(\nabla_Y L \in \mathbb{R}^{N \times d_{\text{out}}}\)

2 計算協方差矩陣

\[\Sigma_{\text{in}} = \frac{1}{N} X^\top X \in \mathbb{R}^{d_{\text{in}} \times d_{\text{in}}}\]
\[\Sigma_{\text{out}} = \frac{1}{N} (\nabla_Y L)^\top (\nabla_Y L) \in \mathbb{R}^{d_{\text{out}} \times d_{\text{out}}}\]

\(\Sigma_{\text{in}}\) 捕捉輸入特徵之間的相關性；\(\Sigma_{\text{out}}\) 反映輸出對損失函數的敏感度。

3 生成白化轉換矩陣

\[S_{\text{in}} = (\Sigma_{\text{in}} + \epsilon I)^{-\frac{1}{2}}\]
\[S_{\text{out}} = (\Sigma_{\text{out}} + \epsilon I)^{-\frac{1}{2}}\]

加入微小常數 \(\epsilon\) 確保數值穩定（論文用 \(10^{-3}\) 作為特徵值截斷閾值）。實作上透過 Cholesky 分解高效計算。

4 對 3D 張量進行「洗白」

\[T_w = T \times_2 S_{\text{out}} \times_3 S_{\text{in}} \tag{4}\]

沿 mode-2（輸出維度）乘上 \(S_{\text{out}}\)，沿 mode-3（輸入維度）乘上 \(S_{\text{in}}\)。

結果：產生一個條件良好的張量 \(T_w\)，所有維度的特徵值被統一為 1.0，相關性被完全消除。

與傳統 2D 白化的差異：傳統方法只對單一矩陣做白化（一個維度）。TD-MoE 的多線性白化同時對輸入和輸出兩個模式進行操作，且作用在整個 3D 張量上——利用多線性結構，顯式適配輸入與輸出的統計幾何。

著色（Re-coloring）：推論零開銷的秘密

白化在壓縮階段帶來巨大效益——但推論時會不會因為多乘白化矩陣而變慢？答案是：完全不會。

問題：白化引入額外矩陣乘法？

Tucker 分解在白化域完成後，原始張量的重建路徑為：

\[T \approx G \times_1 U_1 \times_2 \underbrace{(S_{\text{out}}^{-1} U_2)}_{\text{需要反白化}} \times_3 \underbrace{(S_{\text{in}}^{-1} U_3)}_{\text{需要反白化}}\]

如果每次推論都要乘 \(S^{-1}\)，會增加 \(O(d_{\text{out}}^2 + d_{\text{in}}^2)\) 的額外計算。

解法：預先吸收（Re-coloring）

在壓縮完成後、部署之前，一次性計算：

\[U'_2 = S_{\text{out}}^{-1}\, U_2 \quad\in\mathbb{R}^{d_{\text{out}}\times r_2}\] \[U'_3 = S_{\text{in}}^{-1}\, U_3 \quad\in\mathbb{R}^{d_{\text{in}}\times r_3}\]

然後只儲存 \(U'_2, U'_3\)，丟棄原始的 \(U_2, U_3, S_{\text{in}}, S_{\text{out}}\)。

視覺化：壓縮 vs 推論路徑

計算開銷驗證（A800 GPU）

  • 白化記憶體：Mixtral 僅 890 MB（佔 <1%）
  • 白化 FLOPs：0.11 TFLOPs（一次性）
  • Cholesky：784 MB，即使 FFN 維度達 14,336 也不構成瓶頸
  • 全流程：Mixtral 約 21 秒、Phi-3.5-MoE 約 7 秒

核心數學公式集

校正目標

\[\min_{\{\hat W^{(i)}\}}\;\mathbb{E}_x\!\left[\sum_{i=1}^K\|W^{(i)}x-\hat W^{(i)}x\|^2\right]\tag{1}\]

張量化

\[T\in\mathbb{R}^{K\times d_{\text{out}}\times d_{\text{in}}},\quad T[i,:,:]=W^{(i)}\]

Tucker 分解

\[\min_{G,U_n}\|T_w-G\times_1 U_1\times_2 U_2\times_3 U_3\|_F^2\tag{3}\]

因子矩陣意義

  • \(U_1\!\in\!\mathbb{R}^{K\times r_1}\)：\(r_1\) 個元專家基底
  • \(U_2\!\in\!\mathbb{R}^{d_{\text{out}}\times r_2}\)：輸出子空間
  • \(U_3\!\in\!\mathbb{R}^{d_{\text{in}}\times r_3}\)：輸入子空間

多線性白化

\[T_w = T\times_2 S_{\text{out}}\times_3 S_{\text{in}}\tag{4}\]
\[S_{\text{in}}=(\tfrac{1}{N}X^\top X+\epsilon I)^{-1/2},\quad S_{\text{out}}=(\tfrac{1}{N}(\nabla_Y L)^\top(\nabla_Y L)+\epsilon I)^{-1/2}\]

參數量公式

\[P_{\text{tucker}}=r_1 r_2 r_3 + Kr_1 + d_{\text{out}}r_2 + d_{\text{in}}r_3\]

閉式反解 r₃（預算約束）

\[r_3 = \frac{(1-\rho^*)P_{\text{orig}}-(Kr_1+d_{\text{out}}r_2)}{r_1 r_2+d_{\text{in}}}\tag{5}\]

固定 \((r_1,r_2)\) 掃描，用上式閉式求 \(r_3\)，投影到 \([1,d_{\text{in}}]\)。整個搜尋從三維降為二維。

三維秩分配 + 壓縮地景圖

Algorithm 2 — 秩搜尋

Input: (K, dₒᵤₜ, dᵢₙ), ρ* P_target ← (1−ρ*) × K·dₒᵤₜ·dᵢₙ best_err ← +∞ for r₁ = 1 … K do for r₂ = 1 … dₒᵤₜ do num ← P_target − (K·r₁ + dₒᵤₜ·r₂) den ← r₁·r₂ + dᵢₙ r₃ ← ⌊num/den⌋ if r₃ ∉ [1, dᵢₙ]: skip P_tucker ← r₁r₂r₃ + Kr₁ + dₒᵤₜr₂ + dᵢₙr₃ err ← |P_tucker − P_target| if err < best_err: update Return: (r₁*, r₂*, r₃*)

最內層 \(r_3\) 為閉式 → 搜尋從 \(O(K\!\cdot\!d_{\text{out}}\!\cdot\!d_{\text{in}})\) 降為 \(O(K\!\cdot\!d_{\text{out}})\)。

結語

三大貢獻

  1. 跨專家張量化：首次將 MoE 壓縮提升為整層 3D 聯合分解
  2. 多線性白化：特徵相關降至 <10⁻⁷，大幅改善截斷品質
  3. 三維秩分配：閉式 \(r_3\) 反解，二維掃描精確命中壓縮預算
一句話：TD-MoE 讓「整層一起壓」成為可行且高效的選項，在無微調設定下於高壓縮比大幅領先所有基線。
很適合邊緣設備：在同等品質下顯著減少常駐權重與 VRAM 需求，對記憶體較小的部署環境（單卡、邊緣推理盒、低功耗節點）特別友善。
仍有缺點（工程面）：壓縮後的推論 pipeline（權重佈局、kernel 串接、快取策略）會更難設計，部署系統需額外做算子融合與記憶體排程優化。
