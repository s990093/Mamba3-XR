演化地圖 (The Evolution Map)

今天這篇論文是打破傳統 Attention 運算瓶頸的「萬物起源」。

💡 一句話總結：2020 年的 Linear Attention 是這條演化鏈的「萬物起源」。理解它，就理解了通往 Mamba、GDN 的完整數學脈絡。

萬惡的起源：Softmax Attention｜壹

2017 · Transformer：一層內用 \(q_i,k_j,v_j\) 做軟加權聚合全序列；矩陣式與 \(N^2\)見下頁。

意義

問 \(q_i\) · 比 各 \(k_j\) → softmax 權重 · 取 \(\{v_j\}\) 加權和 · 非因果時可直視全序列（遮罩另加）。

\(Q\)

當步「問什麼」· \(q_i\) · \(Q \in \mathbb{R}^{N \times d_k}\)

\(K\)

被比的「標籤」· \(k_j\) · logit \(= q_i^\top k_j\)

\(V\)

被讀出的內容· \(v_j\) · \(V \in \mathbb{R}^{N \times d_v}\)

↳ 嵌入 → \(Q,K,V\)
\[Q = XW_Q,\; K = XW_K,\; V = XW_V\]

\(X\)：\(N\) 個 token 向量堆疊；Multi-head＝多組 \((W_Q,W_K,W_V)\) 平行再拼。

萬惡的起源：Softmax Attention｜貳·公式與 \(N^2\)

把上一頁的「問答／鍵值」直覺，收斂成一條矩陣式；\(QK^T\) 是長序列下記憶體與計算暴增的第一道關卡。

Standard Softmax Attention
\[\text{Attention}(Q, K, V) = \text{softmax}\!\left(\frac{QK^T}{\sqrt{d_k}}\right) V\]

對第 \(i\) 個位置：softmax 作用在 logits \(\bigl(q_i^\top k_1,\ldots,q_i^\top k_N\bigr)/\sqrt{d_k}\)，得到對 \(\{v_j\}\) 的權重；\(\sqrt{d_k}\) 縮放可避免 \(d_k\) 變大時內積過大、softmax 過尖。

致命瓶頸（為何稱「萬惡」）

必須先形成全配對 logits 矩陣 \(\;QK^T\)，shape 為 \(N \times N\)；序列長度直接平方進複雜度。

  • 時間：\(O(N^2 d_k)\)
  • 空間：\(O(N^2)\)（儲存或反覆進出 attention 區塊）
  • 例：\(N=16\text{k}\)、FP16，單此一塊約 ~1 GB 量級

下一頁起用圖把 \(Q \times K^T \to N \times N\) 的「爆炸」畫出來。

複雜度災難：\(QK^T\) 的代價

矩陣相乘 \(QK^T\) 產生的 \(N \times N\) 矩陣，是所有問題的根源。

💥 二次方的詛咒：當序列長度從 1k 增加到 16k，計算量暴增 256 倍。這使得長文本處理在標準 Transformer 上幾乎不可行。

FlashAttention：硬體優化的極限

在從數學上解決 \(O(N^2)\) 之前，有人嘗試從硬體層級極致優化——但這條路有其天花板。

🔧 FlashAttention 的核心思想

IO-Awareness（記憶體感知）

  • 真正的瓶頸不在算力 (FLOPs)，而在 GPU HBM ↔ SRAM 之間的資料搬移
  • 透過 Tiling（區塊化）技術，將 \(QK^T\) 的計算切成小塊，保留在高速 SRAM 內完成
  • 大幅減少記憶體讀寫次數，實現 2-4× 加速
  • 數學上仍是 Exact Attention（精確計算，非近似）

📊 Linformer 的替代路線

Wang et al. (2020) 證明注意力矩陣本質是低秩矩陣，透過投影矩陣 \(E, F\) 壓縮 Key/Value 維度到固定 \(k\)，達成 \(O(N)\)。但它無法像 RNN 一樣做 \(O(1)\) 推論。

⚠️ 但 FlashAttention 解決不了...

  • \(O(N^2)\) 的數學本質沒有改變 — 計算量依然是二次方
  • 自迴歸推論的 KV Cache 仍然線性增長
  • 序列長度超過硬體極限時依然 OOM
  • 無法在邊緣裝置 (Edge) 部署超長序列
💡 結論：即使有 FlashAttention 這樣的底層系統優化，KV Cache 問題依然無解。因此我們需要從數學架構上根本解決 — 這就是 Linear Attention 的使命。

推論的夢魘：KV Cache 問題

在自迴歸 (Autoregressive) 生成時，每產生一個 token，都需要與所有過去的 KV 重新計算注意力。

KV Cache 機制

  • 生成第 \(i\) 個 token 時，需要 \(K_1, K_2, \ldots, K_i\) 和 \(V_1, V_2, \ldots, V_i\)
  • KV Cache 大小隨序列長度 線性增長
  • 每一步推論的計算量也隨之線性增長
  • 記憶體佔用：\(O(N \cdot d_k)\) per layer
實際影響：LLaMA-2 70B 在 4k context 時，KV Cache 就佔用 ~5GB。擴展到 128k 時完全不可行。

核心解密：Linear Attention 的數學推演

從 Softmax 到 Kernel Trick，一步一步推導 \(O(N)\) 的魔法

Step 1：重新審視 Attention 公式

把矩陣形式拆解為第 \(i\) 個 Query 的求和形式——這是理解一切的關鍵。

1 矩陣形式（我們已知的）
\[\text{Attention}(Q,K,V) = \text{softmax}\!\left(\frac{QK^T}{\sqrt{d_k}}\right) V\]
2 逐元素展開：聚焦第 \(i\) 個 query 的輸出
\[V_i = \frac{\displaystyle\sum_{j=1}^{N} \text{sim}(q_i, k_j) \cdot v_j}{\displaystyle\sum_{j=1}^{N} \text{sim}(q_i, k_j)}\]

變數定義

  • \(q_i \in \mathbb{R}^{d_k}\)：第 \(i\) 個位置的 Query 向量
  • \(k_j, v_j\)：第 \(j\) 個位置的 Key 與 Value
  • \(\text{sim}(q_i, k_j)\)：相似度函數

傳統的 sim 函數

\(\text{sim}(q_i, k_j) = \exp\!\left(\frac{q_i^T k_j}{\sqrt{d_k}}\right)\)

因為有指數函數 \(\exp\)，你無法把 \(q_i\) 從求和中提出來！

插頁：\(QK^\top\) logits 與因果遮罩

自迴歸語言模型裡，第 \(i\) 個 query \(q_i\) 只能與目前與過去的 keys \(\{k_j: j\le i\}\) 計算相似度；未來位置 \(j>i\) 經過遮罩後在 softmax 權重為 0。

CAUSAL SOFTMAX ATTENTION
\[ \mathrm{Attn}(Q,K,V) = \mathrm{softmax}\left(\frac{QK^T}{\sqrt{d_k}} + M\right)\! V \] \[ M_{ij}= \begin{cases} 0 & j \le i \\ -\infty \;\;(\text{masked}) & j > i \end{cases} \]
  • logits：\((QK^T)_{ij} = q_i^T k_j\)。列 \(i\) 對應 \(q_i\)，欄 \(j\) 對應 \(k_j\)。
  • 上三角（\(j>i\)）在加到 \(M\) 後變為 \(-\infty\)，softmax 對應機率為 0，等價於從注意力求和中刪除這些項。

視覺化：\(QK^\top\) 上套用因果遮罩

底色 = softmax 仍可見的有效 logit；灰格 = 被設為 \(-\infty\) 的位置。

有效 \(q_i^\top k_j\) 自注意力（對角） 未來／遮罩

Step 2：Softmax 的限制

指數函數 \(e^{q_i^Tk_j}\) 讓 \(q_i\) 和 \(k_j\) 緊密耦合，阻礙了變數分離。

PROBLEM: INSEPARABLE
\[\text{sim}(q_i, k_j) = \exp\!\bigl(q_i^T k_j\bigr) \neq f(q_i) \cdot g(k_j)\]

為什麼不能分離？

  • 指數函數的特性：\(\exp(a + b) = \exp(a) \cdot \exp(b)\)，但 \(\exp(a^Tb)\) 不能簡單拆解
  • 因為 \(q_i^T k_j\) 是內積，不是簡單的加法
  • 這代表每個 \(q_i\) 都必須和所有 \(k_j\) 配對計算 → \(O(N^2)\)
🤔 關鍵問題：如果我們把 \(\text{sim}(q,k)\) 換成一個可分離的函數呢？這就是 Kernel Trick 的核心思想。

Step 3：Kernel Trick（核方法）

引入特徵映射 (Feature Map) \(\phi(\cdot)\)，替換掉 Softmax。

1 核方法的核心思想：用可分離的內積替代指數函數
\[\text{sim}(q_i, k_j) = \phi(q_i)^T \phi(k_j)\]

❌ 原始 Softmax

\[\exp(q_i^T k_j)\]

\(q_i\) 和 \(k_j\) 不可分離
→ 必須逐對計算 → \(O(N^2)\)

✓ Kernel 形式

\[\phi(q_i)^T \phi(k_j)\]

\(\phi(q_i)\) 和 \(\phi(k_j)\) 獨立計算
→ 可以重新排列運算順序！

📌 Key Insight：只要 \(\phi(\cdot)\) 是一個合法的特徵映射函數，那麼 \(\phi(q_i)^T \phi(k_j)\) 就定義了一個合法的核函數 (Kernel Function)，可以近似任何正定的相似度度量。

Step 4：結合律的魔法 ✨

將 Kernel 形式代入原式——這是整場報告的最高光時刻。

1 代入 Kernel 形式
\[V_i = \frac{\displaystyle\sum_{j=1}^{N} \bigl(\phi(q_i)^T \phi(k_j)\bigr) \cdot v_j}{\displaystyle\sum_{j=1}^{N} \phi(q_i)^T \phi(k_j)}\]
2 關鍵觀察：\(\phi(q_i)^T\) 對求和變數 \(j\) 來說是常數！

→ 因此可以提出求和符號 \(\sum\) 外面

THE MAGIC: ASSOCIATIVE PROPERTY
\[V_i = \frac{\phi(q_i)^T \displaystyle\sum_{j=1}^{N} \phi(k_j) \, v_j^T}{\phi(q_i)^T \displaystyle\sum_{j=1}^{N} \phi(k_j)}\]
🎯 矩陣乘法的結合律：運算順序從 \(\underbrace{(QK^T)}_{N \times N}V\) 變成 \(Q\underbrace{(K^TV)}_{d_k \times d_v}\)。前者產生 \(N \times N\) 矩陣，後者只產生 \(d_k \times d_v\) 矩陣！

Step 5：提取 Query — 降維打擊

把 \(\phi(q_i)\) 提出後，求和內部只剩下與 \(N\) 無關的固定維度。

分子部分

\[\underbrace{\sum_{j=1}^{N} \phi(k_j) \, v_j^T}_{d_k \times d_v \text{ 矩陣}}\]
  • \(\phi(k_j) \in \mathbb{R}^{d_k}\)，\(v_j \in \mathbb{R}^{d_v}\)
  • 外積 \(\phi(k_j) v_j^T\) 為 \(d_k \times d_v\) 矩陣
  • 對 \(N\) 個這樣的矩陣求和 → 結果仍是 \(d_k \times d_v\)
  • ✓ 與序列長度 \(N\) 無關！

分母部分

\[\underbrace{\sum_{j=1}^{N} \phi(k_j)}_{d_k \text{ 向量}}\]
  • \(\phi(k_j) \in \mathbb{R}^{d_k}\)
  • 對 \(N\) 個向量求和 → 結果仍是 \(d_k\) 維向量
  • ✓ 同樣與 \(N\) 無關！
計算一次，對所有 \(i\) 共享。
遍歷 \(N\) 個 token 的成本：\(O(N \cdot d_k \cdot d_v)\)

Step 6：\(O(N)\) 複雜度正式達成 🎉

整體計算複雜度從二次方驟降為線性。

Standard Attention
\(O(N^2 d)\)
先算 \(QK^T\)（\(N \times N\)），再乘以 \(V\)
Linear Attention
\(O(N d^2)\)
先算 \(K^TV\)（\(d \times d\)），再乘以 \(Q\)
比較項目 Standard Attention Linear Attention
時間複雜度 \(O(N^2 d_k)\) \(O(N d_k d_v)\)
空間複雜度 \(O(N^2 + N d_v)\) \(O(N d_k + d_k d_v)\)
中間矩陣大小 \(N \times N\) \(d_k \times d_v\)
當 \(N \gg d\) 時 二次方爆炸 💥 線性增長 ✓

Step 7：特徵映射 \(\phi(x)\) 的選擇

Katharopoulos 等人提出了一個簡單有效的特徵映射函數。

FEATURE MAP
\[\phi(x) = \text{elu}(x) + 1\]

為什麼選 ELU + 1？

  • 非負性保證：\(\text{elu}(x) + 1 > 0\) 對所有 \(x\)，確保分母不為零
  • 計算高效：比 Random Fourier Features 等方法便宜得多
  • 梯度友善：ELU 本身就是常用的激活函數，梯度穩定
  • 簡潔：不需要額外的隨機投影或高維映射

ELU 函數定義

\[\text{elu}(x) = \begin{cases} x & \text{if } x > 0 \\ e^x - 1 & \text{if } x \leq 0 \end{cases}\]

直覺理解：加 1 把 ELU 的值域從 \([-1, +\infty)\) 平移到 \((0, +\infty)\)，保證 Kernel 值恆正。

⚠️ 注意：這個簡單的 \(\phi(x)\) 是 Linear Attention 效能略低於 Softmax 的主要原因之一。後續研究（如 cosFormer、Performer）致力於設計更好的特徵映射。

Step 8：雙重視角 — Training vs. Inference

Linear Attention 的精妙之處：根據使用場景切換計算模式。

🏋️ Training Mode（平行化）

\[V = \phi(Q) \cdot \bigl(\phi(K)^T V\bigr)\]
  • 整個序列一次處理
  • 先算 \(\phi(K)^T V\)（\(d_k \times d_v\)）
  • 再與 \(\phi(Q)\) 相乘
  • 完全平行化，適合 GPU
  • 複雜度：\(O(N d_k d_v)\)

🚀 Inference Mode（遞迴）

\[S_i = S_{i-1} + \phi(k_i) v_i^T\]
  • 逐 token 遞迴更新
  • 維護固定大小的隱藏狀態 \(S_i\)
  • 每步只需一個外積 + 加法
  • 不需要 KV Cache！
  • 複雜度：\(O(d_k d_v)\) per step
🔑 這就是「Transformers are RNNs」的精髓：同一個模型，訓練時像 Transformer（平行化），推論時像 RNN（遞迴）。下一部分，我們正式推導這個 RNN 形式。
Part 3

Transformers are RNNs

將 Linear Attention 無縫轉換為 RNN 的狀態更新形式
— 這是通往現代 SSM/Mamba 的關鍵橋樑

因果遮罩（Causal Masking）的影響

在自迴歸生成中，第 \(i\) 個 token 只能看到過去的資訊。

1 完整的 Attention（雙向）：求和範圍 \(j = 1\) 到 \(N\)
\[V_i = \frac{\phi(q_i)^T \sum_{j=1}^{\color{red}N} \phi(k_j) v_j^T}{\phi(q_i)^T \sum_{j=1}^{\color{red}N} \phi(k_j)}\]
2 因果 Attention（單向）：求和範圍變成 \(j = 1\) 到 \(\mathbf{i}\)
\[V_i = \frac{\phi(q_i)^T \sum_{j=1}^{\color{#0D9488}i} \phi(k_j) v_j^T}{\phi(q_i)^T \sum_{j=1}^{\color{#0D9488}i} \phi(k_j)}\]
💡 核心觀察：因為求和範圍的上界隨 \(i\) 遞增，我們可以把這個求和寫成遞迴形式 —— 「第 \(i\) 步的求和 = 第 \(i{-}1\) 步的求和 + 第 \(i\) 步的新項」。

定義 RNN 的隱藏狀態 (Hidden States)

為了把求和寫成遞迴，我們定義兩個隱藏狀態。

分子狀態 (Numerator State)
\[S_i = \sum_{j=1}^{i} \phi(k_j) \, v_j^T \;\in\; \mathbb{R}^{d_k \times d_v}\]
分母狀態 (Denominator State)
\[Z_i = \sum_{j=1}^{i} \phi(k_j) \;\in\; \mathbb{R}^{d_k}\]

用隱藏狀態表示輸出

\[V_i = \frac{\phi(q_i)^T \cdot S_i}{\phi(q_i)^T \cdot Z_i}\]
直覺理解：數學符號 \(\sum\) 展開的意義
  • 數學符號 \(\sum\) 只是在告訴你「它的總內容是什麼」，公式 \(S_i = \sum_{j=1}^{i}\phi(k_j)v_j^T\) 是為了嚴謹地定義：當我們走到第 \(i\) 步時，這個 \(S_i\) 裡面「究竟包含了哪些東西」。
第 1 步的定義：\(S_1 = \phi(k_1)v_1^T\)
第 2 步的定義：\(S_2 = \phi(k_1)v_1^T + \phi(k_2)v_2^T\)
第 3 步的定義：\(S_3 = \phi(k_1)v_1^T + \phi(k_2)v_2^T + \phi(k_3)v_3^T\)
  • 如果你讓電腦照著定義去算，那第 3 步確實要算 3 次，第 100 步就要算 100 次，這依然是超慢的寫法。
  • 每當新 token 到來，我們必須把這個定義轉成遞迴形式，只需要更新狀態，而非重新計算整個序列。

遞迴更新公式 — 分子 \(S_i\)

把求和拆解為「前一步的結果」加上「新 token 的貢獻」。

1 定義展開
\[S_i = \sum_{j=1}^{i} \phi(k_j) v_j^T = \underbrace{\sum_{j=1}^{i-1} \phi(k_j) v_j^T}_{= \, S_{i-1}} + \;\phi(k_i) v_i^T\]
RECURRENCE — NUMERATOR STATE UPDATE
\[\boxed{S_i = S_{i-1} + \phi(k_i) \, v_i^T}\]

運算分析

  • \(\phi(k_i) \in \mathbb{R}^{d_k}\)，\(v_i \in \mathbb{R}^{d_v}\)
  • 外積 \(\phi(k_i) v_i^T\) → \(d_k \times d_v\) 矩陣
  • 加法 \(S_{i-1} + \cdots\) → 逐元素相加
  • 每步計算量：\(O(d_k \cdot d_v)\) — 常數！

與 RNN 的對照

傳統 RNN：\(h_i = f(h_{i-1}, x_i)\)

Linear Attention：\(S_i = S_{i-1} + g(k_i, v_i)\)

✓ 完全相同的遞迴結構！
狀態 \(S_{i-1}\) 加上新輸入的貢獻。

遞迴更新公式 — 分母 \(Z_i\) 與輸出

分母狀態更新

DENOMINATOR STATE UPDATE
\[\boxed{Z_i = Z_{i-1} + \phi(k_i)}\]

同理，分母的求和也可以寫成遞迴：每步只需加上新的 \(\phi(k_i)\) 向量。

最終輸出

OUTPUT COMPUTATION
\[\boxed{V_i = \frac{\phi(q_i)^T \cdot S_i}{\phi(q_i)^T \cdot Z_i}}\]

用當前 Query 的特徵映射，去「查詢」隱藏狀態中的記憶。

先備知識：什麼是 RNN？

RNN（Recurrent Neural Network）用固定大小的隱狀態 \(h_t\) 在時間上傳遞記憶：每一步讀進新輸入 \(x_t\)，用同一組 參數更新狀態。下一張投影片會用到這個數學「樣版」來對照 Linear Attention。

1 一般形式（抽象）
RECURRENT UPDATE
\[ h_t \;=\; f_\theta\bigl(h_{t-1},\, x_t\bigr),\qquad t = 1,2,\ldots,T \] \[y_t \;=\; g_\theta\bigl(h_t\bigr)\quad \text{（可有可無的讀出）}\]
  • \(h_t\)：時間 \(t\) 的隱狀態（歷史的壓縮摘要）
  • \(x_t\)：時間 \(t\) 的輸入（例如詞向量）
  • 遞迴＝對每個 \(t\) 套用同一規則 \(f_\theta\)：參數不隨長度倍增。
2 經典 Elman／Vanilla RNN（向量式）
\[ h_t \;=\; \sigma\!\left(W_h h_{t-1} + W_x x_t + b\right) \]

\(W_h \in \mathbb{R}^{d_h\times d_h}\)、\(W_x \in \mathbb{R}^{d_h\times d_x}\)、\(b \in \mathbb{R}^{d_h}\)。\(\sigma\) 常見為 \(\tanh\) 或 ReLU（早期文獻也常寫 logistic sigmoid）。

對照上上頁的 Linear Attention

你已經看到 \(S_i = S_{i-1} + \phi(k_i)v_i^T\)、\(Z_i = Z_{i-1}+\phi(k_i)\)——形式上就是「舊狀態 + 可由當步輸入算出的加性增量」；差別在對照項 \(\sigma(W_h h_{t-1}+W_x x_t+b)\)：傳統 RNN 的活化非線性會加重 BPTT 的梯度難題，而這裡的增量是純加法（再配合查表式的分母 \(Z_i\) 與讀出）。

心智圖：因果 Linear Attention = 「用矩陣狀態 \(\;S,Z\;\) 存記憶 + 線性增量」；把它放在 RNN 的框架裡講， 下一頁就是「所以它正是（廣義上）某一類 RNN」的正式對照組。

真正的 RNN 誕生！

Linear Attention 的因果版本，在數學上完全等價於一個 RNN。

傳統 RNN (Elman, 1990)

\[h_i = \sigma(W_h \cdot h_{i-1} + W_x \cdot x_i + b)\]
  • 非線性激活 → 梯度消失/爆炸
  • 隱藏狀態維度受限
  • 無法平行化訓練

Linear Attention RNN (2020)

\[S_i = S_{i-1} + \phi(k_i) v_i^T\] \[V_i = \phi(q_i)^T S_i \,/\, \phi(q_i)^T Z_i\]
  • 線性更新 → 穩定的梯度傳播
  • 矩陣狀態 \(d_k \times d_v\) → 高容量
  • 可同時平行化訓練！
📌 Key Insight：Katharopoulos 等人從數學上證明了——只要替換掉 Softmax 中的指數項，Attention 機制本質上就可以被等價轉換為一種 RNN 的狀態更新。

小結：Linear Attention 的雙重性

一個模型，兩種面貌——這正是它的優雅之處。

🏋️

Training — Transformer 模式

\[V = \phi(Q) \cdot \bigl(\phi(K)^T V_{\text{in}}\bigr)\]
  • 完全平行化的矩陣運算
  • 適合 GPU 大批量訓練
  • 複雜度：\(O(N d^2)\)
🚀

Inference — RNN 模式

\[S_i = S_{i-1} + \phi(k_i)v_i^T\]
  • 逐 token 遞迴狀態更新
  • 固定記憶體、恆定延遲
  • 每步複雜度：\(O(d^2)\)
🎓 「Transformers are RNNs」 — 訓練時是 Transformer，推論時是 RNN。
這不是近似，而是數學上的等價。

反向傳播的挑戰：記憶體退化

如果我們直接對 \(O(N)\) 的 RNN 遞迴更新使用 Pytorch 等框架的 AutoDiff，會發生什麼事？

❌ AutoDiff 的盲點

  • 為了計算梯度，深度學習框架必須保存所有前向傳播的計算圖 (Computation Graph)。
  • 這表示模型必須把每個時間步的隱藏狀態 \(S_1, S_2, \dots, S_N\) 全部存在記憶體中。
  • 每個 \(S_i\) 的大小是 \(d_k \times d_v\)。保存 \(N\) 個狀態 → 記憶體消耗變回 \(O(N \cdot d_k d_v)\)！
  • 這摧毀了我們好不容易達成的固定記憶體優勢。

✓ 解決方案：自定義梯度

  • 放棄自動微分，手動推導並實作反向傳播公式。
  • 我們需要一種算法，能像前向傳播一樣，利用累積和 (Cumulative Sums) 來計算梯度。
  • 目標：反向傳播也要達成 \(O(N)\) 計算複雜度與 \(O(1)\) 額外記憶體。
💡 核心思想：不僅前向推論要 RNN 化，反向傳播（Backpropagation）也必須設計對應的恆定記憶體演算法，Linear Attention 才能真正處理無限長序列。

反向傳播：累積和的奇蹟

Katharopoulos 等人證明了，梯度同樣可以寫成累積和的形式（從 \(N\) 到 \(1\) 反向累積）。

假設 \(\nabla_{\bar{V}_i} \mathcal{L}\) 是損失函數對分子 \(\bar{V}_i = \phi(q_i)^T S_i\) 的梯度。

1. 對 Query 的梯度

\[\nabla_{\phi(Q_i)} \mathcal{L} = \nabla_{\bar{V}_i} \mathcal{L} \left( \sum_{j=1}^i \phi(K_j) V_j^T \right)^T = \nabla_{\bar{V}_i} \mathcal{L} \cdot S_i^T\]
  • 直接利用前向傳播已經算好的 \(S_i\)。
  • 無需額外累積。

2. 對 Key 與 Value 的梯度

\[\nabla_{\phi(K_i)} \mathcal{L} = \left( \sum_{j=i}^N \phi(Q_j) \nabla_{\bar{V}_j} \mathcal{L}^T \right) V_i\]
\[\nabla_{V_i} \mathcal{L} = \left( \sum_{j=i}^N \phi(Q_j) \nabla_{\bar{V}_j} \mathcal{L}^T \right)^T \phi(K_i)\]
觀察紅色項：\(\sum_{j=i}^N (\dots)\)
這是一個反向的累積和！我們只要從序列結尾 \(N\) 一路往回算到 \(1\)，維護一個類似 \(S_i\) 的狀態矩陣即可。完全不需要儲存所有的中間狀態！

Algorithm 1：雙向 Cumulative Sums

結合前向與反向，我們得到了一個極度優雅的 \(O(N)\) 演算法。

▶ 前向傳播 (Forward Pass)

\(S \leftarrow 0\)
for \(i = 1 \dots N\) do
    // 狀態更新
    \(S \leftarrow S + \phi(K_i) V_i^T\)
    // 計算輸出
    \(\bar{V}_i \leftarrow \phi(Q_i) S\)
end
📍 從 \(1 \to N\) 掃描，維護狀態 \(S\)。

◀ 反向傳播 (Backward Pass)

\(\tilde{S} \leftarrow 0\)
for \(i = N \dots 1\) do
    // 狀態更新
    \(\tilde{S} \leftarrow \tilde{S} + \phi(Q_i) \nabla_{\bar{V}_i}\mathcal{L}^T\)
    // 計算梯度
    \(\nabla_{V_i}\mathcal{L} \leftarrow \tilde{S}^T \phi(K_i)\)
    \(\nabla_{\phi(K_i)}\mathcal{L} \leftarrow \tilde{S} V_i\)
end
📍 從 \(N \to 1\) 反向掃描，維護狀態 \(\tilde{S}\)。
🏆 演算法分析：兩次掃描即可完成所有計算，時間複雜度 \(O(N \cdot d^2)\)，空間複雜度 \(O(N \cdot d)\)（只需儲存輸出梯度，無需儲存 \(N\) 個 \(d \times d\) 的中間矩陣）。這徹底解決了長序列訓練的 OOM 問題！

Part 5

啟發與未來

從 Linear Attention 到 SSM 與 GDN —— 通往現代架構的最後拼圖

Linear Attention 的致命傷

這篇論文證明了 Attention = Memory Update。但它有一個致命缺點。

NO FORGET GATE / NO DECAY
\[S_i = S_{i-1} + \phi(k_i) v_i^T\]

❌ 問題

  • 所有過去的資訊都是同等權重加進 \(S_i\) 裡
  • 沒有機制讓模型「遺忘」不重要的過去資訊
  • 狀態矩陣 \(S_i\) 會持續無限累積
  • 長序列下，新資訊被舊資訊「稀釋」

✓ 理想的狀態更新

\[S_i = \underbrace{\gamma \cdot S_{i-1}}_{\text{遺忘舊資訊}} + \underbrace{\phi(k_i) v_i^T}_{\text{加入新資訊}}\]
  • \(\gamma \in (0,1)\) → 指數衰減過去的記憶
  • 允許模型「選擇性保留」重要資訊
  • 類比 LSTM 的 Forget Gate
🧩 缺少的拼圖：如果我們能在 Linear Attention 的 RNN 公式上加入「遺忘機制」，就能解決長序列的記憶退化問題。這正是後續所有研究的起點。

第二階段：記憶機制的進化

Linear Attention 的致命傷是「缺乏遺忘」。接下來的架構都在解決如何管理記憶。

2023 RetNet — 指數衰減
\[S_i = \gamma \cdot S_{i-1} + \phi(k_i) v_i^T\]
  • 引入固定衰減矩陣 \(\gamma\) → 逐漸遺忘過去
  • 定義三種計算模式：Parallel（訓練）、Recurrent（推論）、Chunkwise（長文本）
  • 📌 將 Attention 轉化為「可控記憶體」的重要過渡

[Sun et al., 2023]

2023 Mamba-1 — Selective State Space
\[h_t = \overline{A}(x_t) \cdot h_{t-1} + \overline{B}(x_t) \cdot x_t\]
  • 離散更新 → 連續時間動態系統
  • 衰減係數 \(\overline{A}\) 依賴於輸入（Data-dependent Gating）
  • Selective Scan：模型能選擇性記憶重要資訊
  • 推論速度比 Transformer 快 5×，線性擴展

[Gu & Dao, 2023]

演進脈絡：Linear Attention \(S_i = S_{i-1} + \dots\) → RetNet 加入固定衰減 \(\gamma\) → Mamba 讓衰減依賴輸入 \(\overline{A}(x_t)\)。每一步都讓「記憶管理」更加智慧。

Mamba-2：狀態空間對偶性 (SSD)

Dao & Gu (2024) 在數學上證明了一個驚人的結論：SSM 和 Linear Attention 是同一件事。

STATE SPACE DUALITY (SSD)
\[\text{SSM (Recurrent)} \;\equiv\; \text{Linear Attention (Parallel)} \;\equiv\; \text{Semiseparable Matrices}\]

📐 理論貢獻

  • 三者本質上都是操作同一類「半可分矩陣 (Semiseparable Matrices)」
  • Recurrent（RNN 視角）= Parallel（Attention 視角）= 矩陣分解視角
  • 統一了 Linear Attention、SSM、以及結構化矩陣理論

第三階段：Delta Rule 與 Gated Delta Networks

最前沿的研究：把狀態更新看作「微型神經網路的訓練過程」。

🔄 Delta Rule — 精準記憶更新

\[S_i = S_{i-1} + \beta_i \cdot \phi(k_i)\bigl(v_i^T - \phi(k_i)^T S_{i-1}\bigr)\]
  • 不是盲目地「加入」新資訊，而是先計算預測誤差：\(v_i^T - \phi(k_i)^T S_{i-1}\)
  • 像 SGD 梯度下降一樣修正狀態矩陣
  • [Yang et al., NeurIPS 2024] 解決了 Delta Rule 在 GPU 上跨序列平行化的難題

🚀 GDN — 門控 + Delta Rule

\[S_i = \underbrace{\alpha_i \odot S_{i-1}}_{\text{Gating (清除)}} + \underbrace{\beta_i \cdot \phi(k_i)(v_i^T - \phi(k_i)^T S_{i-1})}_{\text{Delta Rule (精準寫入)}}\]
  • 結合 Mamba-2 的資料依賴門控 + Delta Rule 的精準更新
  • Gating \(\alpha_i\)：負責快速清除不需要的記憶
  • Delta Rule：負責精準寫入新記憶（非盲目累加）
  • 目前 SOTA：Memory Update = Gated Gradient Descent
🏆 最終形態：從 Linear Attention 的簡單加法 \(S_i = S_{i-1} + \phi(k)v^T\)，到 GDN 的「門控梯度下降」——記憶體管理從被動累積演變為主動的、精準的、選擇性的更新。

[Yang et al., 2024] — Gated Delta Networks: Improving Mamba2 with Delta Rule
