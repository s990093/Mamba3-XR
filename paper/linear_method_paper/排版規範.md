### 學術級簡報與文獻 Markdown 排版規範 (NeurIPS / ICLR 風格)

#### 1. 數學公式排版 (Mathematical Equations)

學術寫作中，公式的呈現分為「行內公式」與「獨立區塊公式」，必須嚴格區分以保證閱讀流暢度。

**1.1 行內公式 (Inline Math)**

- **規範：** 當變數、複雜度符號或簡短運算式出現在段落文字中時，使用單一 `$` 符號包覆。
- **範例 (Source)：**
  `在自迴歸推論中，標準 Attention 的時間複雜度為 $O(N^2)$，而透過核方法映射 $\phi(x)$，我們可以將其降至 $O(N)$。`
- **呈現結果：**
  在自迴歸推論中，標準 Attention 的時間複雜度為 $O(N^2)$，而透過核方法映射 $\phi(x)$，我們可以將其降至 $O(N)$。

**1.2 獨立區塊公式 (Display Math)**

- **規範：** 核心推導、遞迴更新公式或佔據較大版面的矩陣，必須使用雙 `$$` 包覆，並獨立成行。若有多行推導，應保持等號對齊。
- **範例 (Source)：**
  ```markdown
  $$V_i = \frac{\phi(q_i)^T \sum_{j=1}^N \phi(k_j) v_j^T}{\phi(q_i)^T \sum_{j=1}^N \phi(k_j)}$$
  ```
- **呈現結果：**
  $$V_i = \frac{\phi(q_i)^T \sum_{j=1}^N \phi(k_j) v_j^T}{\phi(q_i)^T \sum_{j=1}^N \phi(k_j)}$$

**1.3 變數定義清單**

- **規範：** 列出複雜公式後，必須緊接一個無序清單 (Bullet points) 來定義所有出現的符號。
- **範例呈現：**
  其中：
  - $q_i \in \mathbb{R}^{d_k}$ 代表第 $i$ 個位置的 Query 向量。
  - $k_j, v_j$ 分別代表第 $j$ 個位置的 Key 與 Value 向量。
  - $\phi(\cdot)$ 為非線性特徵映射函數，如 $\phi(x) = \text{elu}(x) + 1$。

#### 2. 圖表與架構圖排版 (Figures & Diagrams)

在學術簡報中，圖表必須帶有編號與詳細圖說（Caption），使其具備獨立解讀性。

**2.1 影像置入與圖說格式**

- **規範：** 使用置中的粗體文字標註「Figure 編號」，並使用引用區塊 `>` 來放置圖說，說明該圖欲傳達的核心機制。
- **範例呈現：**

  > **Figure 1: The Associative Property in Linear Attention.** > 透過結合律，計算順序由 $(QK^T)V$ 轉換為 $Q(K^TV)$。左圖展示了標準 Attention $N \times N$ 矩陣運算帶來的記憶體瓶頸；右圖則顯示特徵映射後，維度縮減為固定的 $d_k \times d_v$，從而實現線性複雜度。

#### 3. 演算法與程式碼區塊 (Algorithms & Pseudocode)

當需要展示 Triton Kernel 邏輯、Gating 機制或遞迴狀態更新時，需使用帶有語法高亮的程式碼區塊，並加上清晰的註解。

**3.1 虛擬碼排版**

- **規範：** 註解需明確指出數學公式與程式碼的對應關係。
- **範例 (Source)：**

  ```python
  # Algorithm 1: Recurrent State Update in Linear Attention
  # Initialize hidden states
  S = torch.zeros(d_k, d_v) # Numerator state
  Z = torch.zeros(d_k)      # Denominator state

  for i in range(seq_len):
      # 1. Feature mapping: phi(x) = elu(x) + 1
      k_phi = F.elu(k[i]) + 1.0
      q_phi = F.elu(q[i]) + 1.0

      # 2. State update (RNN form)
      S = S + torch.outer(k_phi, v[i])  # S_i = S_{i-1} + phi(k_i)v_i^T
      Z = Z + k_phi                     # Z_i = Z_{i-1} + phi(k_i)

      # 3. Output computation
      out[i] = (q_phi @ S) / (q_phi @ Z)
  ```

#### 4. 重點強調與文獻引用 (Emphasis & Citations)

**4.1 核心 Insight 強調**

- **規範：** 論文的核心貢獻或結論，請使用粗體字結合表情符號（或獨立的 Blockquote）來吸引視線。
- **範例呈現：**
  **📌 Key Insight:** 作者從數學上證明了，只要替換掉 Softmax 中的指數項，**Attention 機制本質上就可以被等價轉換為一種 RNN 的狀態更新 (State Update)**。

**4.2 文獻引用格式**

- **規範：** 在簡報或筆記的結尾，統一使用帶有連結的清單格式，包含作者、年份、論文名稱與發表會議。
- **範例呈現：**
  **References:**
  1.  [Katharopoulos et al., 2020] _Transformers are RNNs: Fast Autoregressive Transformers with Linear Attention_. ICML.
  2.  [Gu & Dao, 2023] _Mamba: Linear-Time Sequence Modeling with Selective State Spaces_. arXiv.

### 學術級簡報與文獻 Markdown 排版規範 (補充版：圖表、架構與色票)

#### 5. 學術級表格排版 (Academic Tables / Booktabs Style)

頂尖會議的表格有一個不成文的規定：**「絕對不使用垂直線 (No vertical lines)」**，且橫線越少越好，這被稱為 Booktabs 風格。這能最大化閱讀的清爽度。

**5.1 效能與對比表格規範**

- **規範：** 1. 僅保留表頭上下與表格最底部的橫線。2. 數據對齊：文字靠左，數字與數學式置中或靠右對齊。3. **重點高亮：** 在實驗結果中，表現最好的數據必須加粗 (`**`)，表現次佳的可以使用下劃線。
- **範例 (Source)：**
  ```markdown
  | Model                       | Time Complexity | Memory (Inference) | PPL (WikiText) |
  | :-------------------------- | :-------------: | :----------------: | :------------: |
  | Standard Transformer        |    $O(N^2)$     |       $O(N)$       |    **18.5**    |
  | Linformer                   |     $O(N)$      |       $O(1)$       |      21.3      |
  | **Linear Attention (Ours)** |   **$O(N)$**    |     **$O(1)$**     |      19.2      |
  ```
- **呈現結果：**

  | Model                       | Time Complexity | Memory (Inference) | PPL (WikiText) |
  | :-------------------------- | :-------------: | :----------------: | :------------: |
  | Standard Transformer        |    $O(N^2)$     |       $O(N)$       |    **18.5**    |
  | Linformer                   |     $O(N)$      |       $O(1)$       |      21.3      |
  | **Linear Attention (Ours)** |   **$O(N)$**    |     **$O(1)$**     |      19.2      |

#### 6. 架構圖排版與符號語言 (Architecture Diagrams)

在講解從 Attention 變成 RNN 的過程，架構圖的邏輯比文字更重要。學術圖表講求「運算符號化」與「資訊流向明確」。

**6.1 資訊流向與節點設計**

- **規範：**
  1. **方向一致性：** 模型架構的資訊流（Information Flow）必須統一，強烈建議「由下至上 (Bottom-up)」或「由左至右 (Left-to-right)」。
  2. **數學符號化：** 盡量減少圖表內的文字描述，直接使用標準數學運算符號取代。例如：
     - 矩陣相乘使用 $\otimes$ (Tensor Product / MatMul)
     - 逐元素相加使用 $\oplus$ (Element-wise Addition)
     - 特徵映射使用 $\phi(\cdot)$
  3. **維度標註：** 在張量流動的箭頭旁，以小字體標註張量形狀（如 $N \times d_k$），這是硬核系統研究者最看重的細節。

#### 7. 學術簡報專屬色票與視覺風格 (Color Palette & Styling)

學術簡報嚴禁使用高飽和度、刺眼的顏色。我們需要一套能傳遞「數學嚴謹性」與「科技感」，同時兼顧色盲友善（Colorblind-friendly）的配色方案。

**7.1 推薦色票：NeurIPS Deep Tech 沉穩風格**
在繪製架構圖或強調文字時，請嚴格限制在以下 4 到 5 種顏色內：

- **1. 背景色 (Background):**
  - **純淨白 / 淺灰白 (`#FAFAFA` 或 `#FFFFFF`)**：保持版面極度乾淨，不干擾數學公式的閱讀。
- **2. 主文字與主結構色 (Primary Text / Base Structure):**
  - **深石板灰 (`#2D3748` 或 `#1A202C`)**：取代純黑，讓長時間觀看投影片的眼睛更舒適。用於一般推導文字與圖表的外框。
- **3. 核心科技色 / 狀態色 (Tech Blue - 用於標註 $S_i, Z_i$ 等新機制):**
  - **學術靛藍 (`#1E3A8A` 或 `#3182CE`)**：這是一種非常有學術權威感的藍色。適合用來標註 Linear Attention 中的核心改良（例如把 $\phi(k_i) v_i^T$ 上色）。
- **4. 警告與效能瓶頸色 (Alert / Bottleneck - 用於標註 $O(N^2)$ 或 Softmax):**
  - **磚紅色 / 柔和橘紅 (`#E53E3E` 或 `#DD6B20`)**：用來指出傳統 Transformer 的痛點（如記憶體爆炸、複雜度過高）。不要用純紅。
- **5. 輔助/次要資訊色 (Secondary - 用於標註維度或較不重要的層):**
  - **冷灰藍 (`#718096` 或 `#A0AEC0`)**：用於張量維度的標註（如 $N \times d$）或背景網格線。

**7.2 色彩應用實戰範例 (如何在講稿中運用)**

- 當你在解釋公式時，可以利用顏色來引導視線：
  > 「大家請看這個公式，紅色的部分代表傳統的 $\exp(q \cdot k)$ 瓶頸，而藍色的部分是我們替換後的核函數內積 $\phi(q)^T \phi(k)$。」
  > _(在 PPT 上，對應的數學符號就套用上述的磚紅色與學術靛藍)_

```markdown
3. 色彩計畫：全篇維持低飽和度學術風格。

```
