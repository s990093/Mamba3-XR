Cross-Platform Inference: NVIDIA Triton → Apple MLX

We implement a numerically-identical inference stack on Apple Silicon using MLX, enabling deployment on Mac devices without architecture modification.

Cache Management (Hybrid Architecture)

Layer Type Cache Structure Size
Mamba-3 \((h_{\text{state}},\; u_{\text{prev}},\; \theta_{\text{sum}})\) \(H \times N \times P\)
Transformer \((K_{\text{cache}},\; V_{\text{cache}})\) \(H \times L_{\max} \times d_k\)

Per-Layer Compile Strategy

attach_decode_compilation() — Each layer's decode step is individually compiled with mx.compile:
  • Mamba: _decode_step(x, h_s, prev_in, prev_ang)
  • Transformer: _decode_step(x, k_cache, v_cache, seq_pos)
  • LM Head: separately compiled via set_lm_head_compile(True)

Tucker G-Matrix Caching

_G_cache: Expert weight matrices are computed once via \(U_{\text{expert}} \times \mathcal{G}\) and cached. Invalidated only on weight updates. Avoids redundant einsum during autoregressive decoding.
